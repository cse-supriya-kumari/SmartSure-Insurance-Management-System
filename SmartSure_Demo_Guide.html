⚙️ 2. Environment Startup Requirements

  1. MySQL (localhost:3306)
  2. RabbitMQ (localhost:5672)
  3. Redis (localhost:6379)
  4. Zipkin (localhost:9411)

Launch Order in IDE (JDK 17):

  1. Run ConfigServerApplication.java
  2. Run EurekaServerApplication.java (Wait until started)
  3. Run ApiGatewayApplication.java
  4. Run AuthService, PolicyService, ClaimsService, and AdminService

🚦 3. Swagger UI Demo Playbook

  • Auth Swagger: http://localhost:8081/swagger-ui/index.html
  • Policy Swagger: http://localhost:8082/swagger-ui/index.html
  • Claims Swagger: http://localhost:8083/swagger-ui/index.html
  • Admin Swagger: http://localhost:8084/swagger-ui/index.html

Step 1: Admin Login & Product Creation

1. Open Auth Service Swagger.
2. Navigate to POST /api/auth/login.

{
  "email": "admin@smartsure.com",
  "password": "adminpassword"
}

3. Copy token, click Authorize -> Bearer YOUR_TOKEN.
4. Open Admin Service Swagger.
5. Navigate to POST /api/admin/policies.

{
  "name": "Super Health Plus",
  "description": "Complete medical coverage",
  "basePremium": 15000.0,
  "coverageAmount": 1000000.0,
  "durationMonths": 12
}

Step 2: Customer Registration & Login

1. Open Auth Service Swagger -> POST /api/auth/register.

{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "password": "password123",
  "phone": "9988776655",
  "address": "123 Smart Ave"
}

2. Call login endpoint with Jane's credentials. Copy token. Authorize on Policy and Claims Swagger.

Step 3: Purchasing a Policy

1. **Policy Service Swagger** -> POST /api/policies/purchase.

{
  "userId": 2,
  "policyTypeId": 1
}

Step 4: Initiating a Claim

1. **Claims Service Swagger** -> POST /api/claims/initiate.
2. Add Header: Idempotency-Key : claim-req-001

{
  "policyId": 1,
  "userId": 2,
  "description": "Hospitalization for Dengue"
}

Step 5: Admin Claim Review & Approval

1. **Admin Service Swagger** -> GET /api/admin/claims/pending.
2. **Admin Service Swagger** -> PUT /api/admin/claims/1/review.

{
  "status": "APPROVED",
  "remarks": "Documents verified."
}

🌟 4. Showcasing the 5 Advanced Patterns

1) Distributed Tracing (Zipkin + Micrometer)

Open http://localhost:9411. Show tracing for one request jumping across Gateway -> Admin -> Claims.

2) RabbitMQ Audit Trail (Event-Driven)

Open http://localhost:15672 to show Queues. Check MySQL `admin_audit_logs` table to see async generated records.

3) Circuit Breaker (Resilience4j)

Stop Claims Service in Eclipse. Run GET /api/admin/reports in Admin Swagger. Show the graceful fallback message: "One or more services are currently unavailable. Cannot generate reports."

4) CQRS with Redis Caching

Run GET /api/admin/reports/dashboard repeatedly in Admin Service. Note response times dropping to ~2ms due to Redis.

5) Idempotency Interceptor

Call POST /api/claims/initiate with header Idempotency-Key: test-123 twice. Show that the second call returns HTTP 409 Conflict!
