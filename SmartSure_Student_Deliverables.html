3. Sequence Diagram — Policy Purchase Flow

This sequence diagram captures the end-to-end flow when a customer purchases an insurance policy, including JWT validation via Feign, database persistence, and asynchronous event publishing.

sequenceDiagram actor Customer participant GW as API Gateway :8080 participant PS as Policy Service :8082 participant AS as Auth Service :8081 participant DB as policy_db MySQL participant MQ as RabbitMQ Broker Customer->>GW: POST /api/policies/purchase Note right of Customer: Headers: Authorization, Idempotency-Key GW->>PS: Route via Eureka Note over PS: IdempotencyInterceptor check PS->>AS: Feign: validateToken() Note over PS,AS: CircuitBreaker enabled AS-->>PS: UserSummaryDTO alt Role = CUSTOMER or ADMIN PS->>DB: Find PolicyType by ID DB-->>PS: PolicyType entity PS->>DB: INSERT Policy (ACTIVE) DB-->>PS: Policy saved PS->>DB: INSERT Premium installments (PENDING) DB-->>PS: Premiums saved PS->>MQ: Publish PolicyPurchasedEvent PS-->>GW: 200 PolicyDTO GW-->>Customer: Policy Details + Premium Schedule else Unauthorized PS-->>GW: 401 Unauthorized GW-->>Customer: Error Response end Note over MQ: Async: Admin Service consumes event and creates AdminAuditLog

Key Design Patterns in This Flow

PatternImplementation
IdempotencyIdempotencyInterceptor checks Idempotency-Key header - prevents duplicate charges (returns 409 on retry)
Circuit BreakerResilience4j @CircuitBreaker(name="authService") with purchasePolicyFallback() method
Event-DrivenPolicyPurchasedEvent published to RabbitMQ, consumed by Admin Service for audit logging

This diagram covers the complete claim lifecycle across three phases: initiation by customer, document upload, and admin review/approval.

sequenceDiagram actor Customer actor Admin participant GW as API Gateway :8080 participant CS as Claims Service :8083 participant PS as Policy Service :8082 participant DB as claims_db MySQL participant FS as File Storage participant MQ as RabbitMQ Broker rect rgb(230, 245, 255) Note over Customer,MQ: Phase 1 — Claim Initiation Customer->>GW: POST /api/claims/initiate GW->>CS: Route request CS->>PS: Feign: getPolicyById() Note over CS,PS: CircuitBreaker enabled PS-->>CS: PolicyDTO CS->>DB: INSERT Claim (SUBMITTED) CS->>MQ: Publish ClaimSubmittedEvent CS-->>Customer: ClaimDTO end rect rgb(255, 245, 230) Note over Customer,MQ: Phase 2 — Document Upload Customer->>GW: POST /api/claims/{id}/upload Note right of Customer: multipart/form-data GW->>CS: Forward file CS->>FS: Store file on disk FS-->>CS: File path CS->>DB: INSERT ClaimDocument CS->>DB: UPDATE status UNDER_REVIEW CS->>MQ: Publish ClaimStatusChangedEvent CS-->>Customer: ClaimDocumentDTO end rect rgb(230, 255, 230) Note over Admin,MQ: Phase 3 — Admin Review Admin->>GW: GET /api/admin/claims/pending GW-->>Admin: List of pending claims Admin->>GW: GET /api/admin/claims/{id}/documents GW-->>Admin: Claim documents Admin->>GW: PUT /api/admin/claims/{id}/review Note right of Admin: {status: APPROVED, remarks: "..."} GW->>CS: Update status via Admin Service CS->>DB: UPDATE status APPROVED CS->>MQ: Publish ClaimStatusChangedEvent CS-->>Admin: Updated ClaimDTO end Note over MQ,DB: Async: All events logged to AdminAuditLog

Container Startup Order

  1. MySQL, RabbitMQ, Redis — Infrastructure (no dependencies)
  2. Eureka Server — Waits: none (healthcheck ready)
  3. Config Server — Waits: Eureka healthy
  4. Auth Service — Waits: MySQL + Eureka + Config healthy
  5. Policy Service — Waits: MySQL + RabbitMQ + Eureka + Config + Auth started
  6. Claims Service — Waits: MySQL + RabbitMQ + Eureka + Config + Policy started
  7. Admin Service — Waits: all infrastructure + all business services
  8. API Gateway — Waits: Eureka + Config + all business services started

Appendix B: Design Patterns Summary

PatternImplementationService(s)
Circuit Breaker Resilience4j @CircuitBreaker with fallback methods Policy, Claims
Idempotent API IdempotencyInterceptor using Idempotency-Key header Policy, Claims
Distributed Caching Redis-backed dashboard and report generation (<5ms on cache hit) Admin
Event-Driven Auditing RabbitMQ pub/sub → AdminAuditLog table Policy, Claims → Admin
Service Discovery Netflix Eureka-based registration & dynamic lookup All services
Centralized Config Spring Cloud Config Server for externalized configuration All services
API Gateway Spring Cloud Gateway with route-based request routing API Gateway
JWT Security Spring Security + custom JwtUtil for token generation/validation Auth, Gateway
Distributed Tracing Micrometer Tracing with Zipkin export All services
